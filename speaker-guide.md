# PyWaw Speaker Guide 🎤

Welcome! We're always looking for speakers — whether it's your first talk ever or your hundredth.

## Talk Formats

| Format | Duration | Best For |
|--------|----------|----------|
| **Lightning talk** | 5-10 min | Quick tips, tool demos, personal projects |
| **Regular talk** | 20-30 min | Deep dives, tutorials, case studies |
| **Workshop** | 45-60 min | Hands-on coding sessions |

## Topic Ideas

Anything Python-related! Some popular categories:

- 🔧 **Tools & Libraries** — New packages, frameworks, dev tools
- 🏗️ **Architecture** — Design patterns, best practices, scaling
- 🤖 **AI/ML** — Machine learning, LLMs, data science with Python
- 🌐 **Web** — Django, FastAPI, Flask, async web
- 📊 **Data** — Pandas, visualization, ETL pipelines
- 🧪 **Testing** — pytest, TDD, CI/CD
- 💼 **Career** — Transitioning to Python, interview prep, freelancing
- 🐣 **Beginner-friendly** — "Things I wish I knew when starting Python"

## Tips for First-Time Speakers

1. **Start small** — A 10-minute lightning talk is perfect for your first time
2. **Tell a story** — "I had this problem → I tried this → Here's what I learned"
3. **Show code** — Live demos beat slides every time
4. **Practice** — Run through it at least twice. Time yourself.
5. **It's OK to be nervous** — Everyone is. The audience is rooting for you.
6. **Ask for feedback** — We'll help you prepare if you want

## Submitting a Talk

1. Email us or reach out on Meetup
2. Include: title, short description (2-3 sentences), estimated duration
3. We'll schedule you for the next available meetup
4. No formal review process — if you want to talk, we want to hear you

## Technical Setup

- Projector with HDMI (bring adapter if needed)
- WiFi available
- We can record talks (optional, with your permission)
